Design notes · Concept A

  • Hierarchy: Rank → Symbol → Name → % (price optional on mobile).
  • Row action: Whole row + text link “Why it's moving” → Deep Dive (Overview tab).
  • No teaser line on v1—keeps rows to two visual lines on phone.
  • Dashboard link: Wheel shows top 5; footer “See all movers” lands here.
  • Risk: Can feel sparse vs premium FORGENIQ wheel—compensate with typography, not widgets.
Concept B

Bloomberg-style movers board

Dense, scannable grid for users who want “market tape” energy without jargon. Column headers in plain English; mini change bars instead of IV/greeks. Still beginner-safe if labels stay human (“Change today”, not “Chg %”).

High information density Pro-adjacent · not pro-only Session mood strip Underline tabs (terminal cue)

Design notes · Concept B

  • Mood strip reuses wheel/session copy—one line, not a second story panel.
  • Change bars visualize magnitude without “relative volume” labels.
  • Row → Deep Dive on click; no inline WIM. Optional keyboard row focus.
  • Beginner guardrail: Hide “Vol”, “RVol”, “Sector” until v1.1; keep 5–6 columns max.
  • Risk: Can intimidate if headers slip into terminal slang—copy review is critical.
Concept C

Retail-first card layout

Robinhood / Cash App energy: each mover is a card with a plain-English teaser and a gold CTA. Best when you want emotion and story hints on the list itself—still opens Deep Dive for full analysis.

Highest approachability Teaser from MOVER_STORIES Summary pills (up / down / active) More vertical scroll on mobile

Design notes · Concept C

  • Teaser = truncated MOVER_STORIES.lead (max ~2 lines)—bridges Dashboard story panel and full Deep Dive.
  • CTA button is explicit for beginners; card tap also opens Deep Dive.
  • Top pills answer “how many up/down?” before scrolling—helps 3-second comprehension.
  • Discover Stocks footer reinforces IA split (today’s tape vs find ideas).
  • Risk: Longest page, slowest scan for “just show me the list”; cap teasers strictly.
